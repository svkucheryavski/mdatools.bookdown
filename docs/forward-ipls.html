Getting started with mdatools for R

0.1 Forward iPLS

The forward approach works as follows

  1. Split spectral data into \(N\) intervals
  2. Create an empty vector with selected intervals
  3. Create a model where intervals in the vector (already slected) are combined with one of the rest. If combination improves the model, add the this new interval to the vector.
  4. Repeat previous step until there is no improvements.

Apparently when nothing is selected, on step 3, the algorithm simply create model for every interval available. As you can see iPLS is iterative procedure. If you have 20 intervals you need to create 20 models on the first step, 19 models on the second step and so on.

In mdatools, you can run iPLS selection by using function ipls. You have to specify matrix with predictors (x) and responses (y), number of components in global model (glob.ncomp), and define intervals. In addition to that you can also specify parameters of cross-validation (cv) as well as criterion for selection of optimal number of components (ncomp.selcrit) for local models similar to PLS. Forward iPLS is used as a default algorithm.

Intervals can be defined using one of the three parameters: either number of intervals (int.num), width of an interval (int.width) or specify start and end of each interval as a two-column matrix with variable indices (int.limits). In addition to that, you can specify maximum number of component for local models (int.ncomp) and maximum number of iterations — so method will stop regardless you have improvements or not (int.niter). By default the number of maximum itrations is limited to 30.

Here is an example of using iPLS for Simdata (for concentration of third chemical component, C3)using 30 intervals.

data(simdata)
X = simdata$spectra.c
y = simdata$conc.c[, 3]

m = ipls(X, y, glob.ncomp = 4, int.num = 30)

As you can see, by default method shows information for every step on the console. Use parameter silent = TRUE to change this behaviour. From the example above we can see that the global model had \(RMSECV = 0.005880\) with 3 components. Creating local models with individual intervals gave the best performance with \(RMSECV = 0.013965\) (interval #9). Combination of the interval #9 with interval #13 (next step) gave \(RMSECV = 0.006588\). The small improvement was obtained for adding 18 more intervals giving final result with \(RMSECV = 0.005765\).

Also you can see the first step (performance of individual models) and the selected interval by using function plot() for the whole model:

plot(m)

You can also see the improvements of RMSECV graphycally:

plotRMSE(m)

Dashed line in this case shows RMSECV value for the global model with all intervals included.

Fnally you can get both seleced intervals and corresponding indices of variables (all of them not just interval limits) as follows:
